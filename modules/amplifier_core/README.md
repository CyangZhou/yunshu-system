# Amplifier Core (云舒增幅器主流程)

## 1. 模块作用
`amplifier_core` 是云舒系统的核心中枢，负责接收上层任务请求，调度并统筹代码生成、文档编写等增幅工作流。它是串联研究归档、回归测试与上下文管理的桥梁。

## 2. 实现原理
采用纯脚本驱动机制，通过读取外部环境上下文与传入的 JSON 参数，解析系统增幅目标，并将其分解为多个子步骤依次执行。整个过程遵循"先规划、后执行"的原则，确保高复杂度工程任务能够被稳定地分步攻克。

## 3. 实现方法
通过全局路由 `scripts/main.py` 的 `--action amplifier_core` 参数触发当前目录下的 `runner.py`。
`runner.py` 内部实现包括：
- 参数与上下文校验
- 工作流动态编排（依赖图构建）
- 任务执行引擎（执行代码重构、文件创建等）
- 执行结果聚合与标准化日志输出

## 4. AI 制作与维护思路 (自解释机制)
- **清晰的代码边界**：保持函数职责单一，逻辑分层，以便 AI 代理在后续迭代时能精准定位。
- **详尽的自我说明**：重要执行步骤必须配备详尽的中文注释，解释"为什么这样做"。
- **防黑盒的容错处理**：捕获所有异常并输出带有排查建议的日志，避免系统静默崩溃，使 AI 能够依据日志自主修复。

## 5. 指令集
可通过以下 CLI 指令唤起本模块：
```bash
python .trae/skills/yunshu-system-skill/scripts/main.py --action amplifier_core --root "." --input '{"task_id":"A001","target":"系统核心架构重构"}'
```
