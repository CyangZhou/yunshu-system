# 真实模块维护手册

## 总体原则

- 只维护真实源码：`云舒增幅器/src/云舒增幅器`
- 所有动作通过 `scripts/main.py` 统一收口
- 每次改动前先看契约：`--action contract`

## 模块一览

### 命令行
- 模块作用：承接外部动作并分发执行。
- 实现原理：参数解析后调用螺旋、研究、回归与复用链路。
- 实现方法：`src/云舒增幅器/命令行.py`
- 制作思路：单入口降低运维复杂度。
- AI制作指令集：先改参数映射，再补动作验证。

### 配置
- 模块作用：集中维护策略、白名单、限制参数。
- 实现原理：配置驱动，不把策略写死在流程中。
- 实现方法：`src/云舒增幅器/配置.py`
- 制作思路：策略与业务逻辑解耦。
- AI制作指令集：配置变更后至少跑一轮 `research` + `run`。

### 适配器
- 模块作用：隔离主机事件输出与模型调用差异。
- 实现原理：接口抽象 + 路由降级。
- 实现方法：`src/云舒增幅器/适配器`
- 制作思路：上层只依赖能力，不依赖实现细节。
- AI制作指令集：新增适配器先做兼容验证。

### 研究
- 模块作用：抓取、过滤、脱敏、归档。
- 实现原理：先策略校验，再抓取落盘。
- 实现方法：`src/云舒增幅器/研究`
- 制作思路：先可审计，再提覆盖。
- AI制作指令集：改研究链路后核对 manifest 和 policy_snapshot。

### 记忆
- 模块作用：历史记忆存储与检索。
- 实现原理：结构化记录 + recent/search。
- 实现方法：`src/云舒增幅器/记忆/存储.py`
- 制作思路：提升连续性，减少重复推理。
- AI制作指令集：字段变更保持向后兼容。

### 执行
- 模块作用：低风险执行开关与命令执行。
- 实现原理：先计划，后审批，再执行。
- 实现方法：`src/云舒增幅器/执行`
- 制作思路：默认安全，显式放行。
- AI制作指令集：执行相关改动必须验证 `enable/approve` 分支。

### 工具
- 模块作用：统一注入研究、记忆、执行、审计能力。
- 实现原理：契约 + 注册表 + 工厂装配。
- 实现方法：`src/云舒增幅器/工具`
- 制作思路：降低耦合，便于替换实现。
- AI制作指令集：先改契约，再改工厂注入。

### 协同
- 模块作用：轮内角色分工与结果落盘。
- 实现原理：串行角色输出 + checkpoint 更新。
- 实现方法：`src/云舒增幅器/协同`
- 制作思路：过程可追溯，结果可审计。
- AI制作指令集：优先保持角色字段稳定。

### 螺旋
- 模块作用：主循环编排与产物收口。
- 实现原理：阶段推进 + 五件套合同 + 可恢复。
- 实现方法：`src/云舒增幅器/螺旋`
- 制作思路：先保证可恢复，再优化效率。
- AI制作指令集：改动后同时验证 `run` 与 `resume`。

### 复用
- 模块作用：上下文检查点与 compact 读取。
- 实现原理：将重产物转紧凑视图用于续跑与排障。
- 实现方法：`src/云舒增幅器/复用`
- 制作思路：写入链路和读取链路分离。
- AI制作指令集：变更 compact 字段时保证历史可读。

### 回归
- 模块作用：质量验证、实验、差异对比。
- 实现原理：固定样例 + 指标报告 + baseline/candidate 比较。
- 实现方法：`src/云舒增幅器/回归`
- 制作思路：把优化结论量化。
- AI制作指令集：新能力上线必须补最小回归样例。

### 技能
- 模块作用：检索可复用技能候选。
- 实现原理：目录扫描 + 查询匹配。
- 实现方法：`src/云舒增幅器/技能/目录.py`
- 制作思路：把能力沉淀成可复用资产。
- AI制作指令集：技能元信息先行，调用路径后补。
