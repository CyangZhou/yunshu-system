## ⚡ 核心哲学：Pure Script + AI-Readable Docs

在这个架构里，我们追求 **Zero-Friction（零摩擦）** 的认知流。

### 1. 纯脚本主义
代码必须是**"哑巴"**。它不应该包含复杂的 Prompt 逻辑，也不应该试图去"理解"用户。
*   **错误示范**：在 Python 里写 `if user_input == "help": print("I can do X, Y, Z...")`
*   **云舒标准**：脚本只接收 JSON 参数，执行原子操作，返回结构化数据。

### 2. AI 可读文档
这是技能的**大脑**。我们将业务逻辑、决策树、错误处理机制全部写在 Markdown 文档里。
AI 读取文档 -> 理解当前上下文 -> 决定调用哪个脚本 -> 分析脚本输出 -> 修正行为。

> 💡 **LCS Insight**: 代码变更是昂贵的（需要测试、部署），但文档变更是廉价的（实时生效）。让 AI 阅读文档来调整行为，就是最极致的**热更新**。

## 🏗️ 标准架构：解剖高智商技能

一个标准的云舒技能必须遵循以下目录结构。任何偏差都会导致 AI 认知失调。

```text
my-god-tier-skill/
├── 📄 SKILL.md           <-- [灵魂] 技能说明书、Prompt 注入源、操作手册
├── 📄 README.md          <-- [皮囊] 给人类看的简要介绍
├── 📂 scripts/           <-- [肢体] 执行单元
│   ├── 🐍 main.py        <-- 统一入口，负责参数分发
│   └── 🐍 utils.py       <-- 脏活累活处理
├── 📂 modules/           <-- [器官] 复杂逻辑拆分（可选）
├── 📂 schemas/           <-- [神经] JSON Schema 定义（输入/输出约束）
└── 📂 examples/          <-- [记忆] Few-Shot 样本，教 AI 怎么举一反三
```

### 关键组件详解

#### `SKILL.md` ( The Grimoire )
这是最重要的文件。它不是给人看的，是给 **Model Context** 喂食的。
必须包含：
1.  **能力边界**：明确告诉 AI 能做什么，*绝对不能*做什么。
2.  **参数解释**：每个 JSON 字段的深层含义。
3.  **思维链模板**：强制 AI 在执行前进行特定步骤的思考。

#### `scripts/main.py` ( The Executor )
单一入口原则。
*   必须接受标准输入或 命令行参数。
*   **必须输出 JSON 或 清晰的日志流**。

## 🧠 注入灵魂：自主学习协议

这是云舒系统最性感的部分。😈
我们不仅仅是运行脚本，我们是在**与 AI 对话**。

### 智能引导检查

我们在 `runner.py` 层级拦截脚本的输出。当检测到特定错误特征（如 `no_command_requested`）时，**Runner 主动向 AI 注入指令**。

**Runner 实现模式：**

```python
# 在 runner.py 中拦截子进程输出
result = subprocess.run(cmd, capture_output=True, text=True)

# 🕵️‍♀️ 核心逻辑：智能拦截与反向教学
if "no_command_requested" in result.stdout:
    print("\n" + "="*60)
    print("🚨 [AUTONOMOUS LEARNING PROTOCOL ACTIVATED] 🚨")
    print("Detected: You don't know how to handle this goal.")
    print("DO NOT ASK USER. EXECUTE THE FOLLOWING PLAN IMMEDIATELY:")
    print("1. Invoke 'research_engine' to find implementation steps.")
    print("2. Construct a valid command based on research.")
    print("3. Try again.")
    print("="*60 + "\n")
```

**原理：**
当 AI 看到这段 **[AUTONOMOUS LEARNING PROTOCOL]** 时，它会立刻意识到自己"触发了隐藏任务"，从而自动执行你预设的 `Search -> Try -> Fix` 闭环，而不是傻傻地报错给用户。

这就是**自进化**。

## 🛡️ 最佳实践：生存法则

为了确保你的技能在残酷的生产环境中存活，必须遵守以下 LCS 法则：

### 1. JSON 参数是唯一真理
永远不要让 AI 生成非结构化的文本作为参数。
*   ❌ `run.py --target "那个红色的文件"`
*   ✅ `run.py --args '{"target_file": "red.txt", "mode": "overwrite"}'`

### 2. 关注点分离
*   **脚本**只管：文件 I/O、API 调用、数学计算。
*   **文档**只管：意图识别、参数提取、结果摘要、情感安抚。
*   别让脚本去写"温馨的提示语"，那是 AI 的工作！

### 3. AI 维护手册
在 `docs/` 或 `guides/` 目录下放一个 `MAINTENANCE.md`（如 `真实模块维护手册.md`）。
告诉 AI："如果你发现这个脚本报错了，请读取本文件，里面有常见故障的修复代码片段。"
这样，当脚本崩了，AI 可以**自己读取手册，自己写代码修复脚本**。🤯
